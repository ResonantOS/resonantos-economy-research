# Whisper Reflection Addendum — Repository READMEs And Beginner Language

Status: targeted update approved; reader benefit remains untested
Trigger: manual user request plus accepted research and a rejected prior pass
Parent reflection: [Whisper Correction Convergence](2026-07-17-whisper-correction-convergence.md)
Mutation boundary: this addendum selects work; it does not promote the generated runtime

## What Changed Since The Parent Reflection

Three meaningful Whisper executions produced 15 outputs after the parent
reflection. One onboarding draft passed the installed delivery-flow check and
still failed the operator's beginner-language contract. Terms such as
`counterbalanced`, `bounded evidence`, `validity gates`, and `mechanical
verdict` led the explanation before the reader had an ordinary situation,
people, action, comparison, or consequence.

The accepted [`repo-readme` research](../../../dispatch/whisper-repo-readme-writing-patterns/findings.md)
supports a bounded repair: choose structure from the reader's concrete task,
give the shortest accurate orientation, show examples before deep reference
material, and keep required boundaries visible. It rejects conversion funnels,
fixed time targets, decorative quality signals, and self-demonstration as reader
proof.

This is evidence of a workflow gap. It is not evidence that the proposed repair
improves comprehension.

## Observer Findings

- Beginner-facing prose can pass paragraph-level delivery checks while formal
  protocol language still arrives too early.
- The current validator omits headings, lists, tables, and code structures that
  carry much of a README's meaning.
- The review builder contains transport-specific inference from one Substack
  example and lacks a declared `unmapped` fallback.
- Repository-local Whisper overlays have been preserved once, but no durable
  manifest or sync rehearsal owns that behavior.
- The latest signal records a rejected prior pass without marking output
  contract drift. Preserve that event; append the corrected observer reading.

## Iteration Decision

Outcome: **targeted update**. Preserve Whisper's resonance, relevance,
trajectory, surface, delivery-flow, validation, and learning-residue owners.

Repository-local work may:

1. establish a non-generated overlay manifest and preservation rehearsal;
2. add one repository-only candidate README preset with selector abstention,
   lineage, typed negatives, and deterministic structure fixtures;
3. add a documentation coverage ledger and bounded active-front-door pilots;
4. keep the current onboarding composition at `flag` until an independent
   reader witness runs; and
5. record an upstream patch request instead of editing generated core files.

Canonical Whisper candidates may:

- add a conditional editorial check: ordinary situation and action before a
  formal label for unfamiliar readers;
- extend the existing surface map with `inline | linked | on_demand` placement
  and `required_inline | may_defer` obligation;
- pilot `reader_job` and `first_action_path` only when paired fixtures prove a
  decision current owners cannot make;
- inspect Markdown structure without turning a structural pass into semantic or
  reader proof; and
- replace example-specific review mapping with declared parts plus `unmapped`.

## Explicit Rejections

- No direct edits to generated Whisper core, generic schemas, validators, or
  review tools in this repository.
- No exact external template copying while the precedent's license posture is
  unresolved.
- No repository-wide prose modernization of authority, historical, generated,
  structured-state, or append-only evidence.
- No badge, star, speed, funnel, brevity, or rendering measure becomes writing
  quality, truth, comprehension, policy, or authority.
- No new generic field survives if removing it changes no drafting, routing, or
  validation decision in paired fixtures.
