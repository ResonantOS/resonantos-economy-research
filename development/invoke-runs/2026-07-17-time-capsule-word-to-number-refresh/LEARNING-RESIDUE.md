# Whisper Learning Residue — Mechanism Clarity

## Correction Learned

The audience did not primarily need a more inviting explanation of the time
capsule. It needed a visible causal account of how symbolic input becomes a
bounded numeric result.

## Durable Lessons

1. **A safe abstraction can still be a black box.** “Shared pattern” avoided
   false claims but did not explain the mechanism.
2. **Name each type change.** Word → record → fingerprint → range result is more
   understandable than word → number.
3. **Explain the second input's job.** “Later public value” matters because it
   keeps the final result unavailable while contributions are open.
4. **Show invariants during comparison.** Holding the fixture records, future
   value, range, and rules fixed lets one changed word demonstrate its actual
   causal role.
5. **Do not trade clarity for false simplicity.** Letter sums are easy to teach
   but not the mechanism and invite strategic manipulation.
6. **Computed examples are claims.** Every displayed hash and result must come
   from declared fixtures and a reproducible algorithm.
7. **Presence is not comprehension.** A phrase appearing in copy cannot pass a
   human understanding gate.

Implement the transformation explorer locally, reproduce its fixture result,
then test whether first-time readers can explain the complete chain without
