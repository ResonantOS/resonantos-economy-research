# Whisper Composition Plan — Mechanism Clarity

Status: `pass`

## Candidate Set

### A — Letter Arithmetic (`rejected`)

Map letters to integers and add the results.

- Resonance: high; people can picture it quickly.
- Relevance: superficially high.
- Trajectory: simple.
- Fatal cost: it is not the selected mechanism, rewards spelling/length
  strategies, and suggests a fixed token value per word.

### B — Shared Pattern (`rejected as primary explanation`)

Say all words become one shared pattern that helps choose the number.

- Resonance: warm and collective.
- Relevance: preserves mechanism boundaries.
- Trajectory: too compressed.
- Fatal cost: it hides the exact transformation the audience is asking about.

It may survive only as a secondary metaphor after the record/fingerprint chain
is understood.

### C — Visible Transformation (`selected`)

Show word → record → whole-capsule fingerprint + later public value → number in
the pre-agreed range, then let a local invented example compute both a baseline
and one hypothetical comparison.

- Resonance: preserves the sense that one small word changes a shared whole.
- Relevance: directly answers the reported question.
- Trajectory: each step causes the next.
- Cost: introduces more mechanism language, controlled through layered detail.

### D — Cryptographic Proof First (`rejected for first contact`)

Lead with canonical JSON, SHA-256, Merkle roots, beacon bytes, and rejection
sampling.

- Accuracy: high.
- Reproducibility: high.
- Relevance to a verifier: high.
- Fatal cost: asks general readers to understand implementation before the
  causal model.

Retained as Layer 3 proof detail.

## Pareto Selection

Candidate C is non-dominated across causal clarity, claim safety, emotional
meaning, and implementation cost. A is easier but false; B is warmer but opaque;
D is precise but cognitively expensive.

## Proposer / Balancer — Round 1

### Proposer

Lead with “Your word changes the capsule. The capsule changes the number.”

### Balancer

The line is memorable but still skips the transformation. “Changes” could mean
voting, summing, scoring, or symbolism.

### Reconciliation

Use the exact reader question as the heading, then answer in four causal
sentences and show the pipeline immediately beneath it.

## Proposer / Balancer — Round 2

### Proposer

Let the person edit a word and watch the result change.

### Balancer

Editing after the later value appears could teach the opposite of the frozen
timeline. Decorative hashes would also teach trust without proof.

### Reconciliation

Show two hypothetical capsules side by side. Keep the invented community
records, future value, range, and rules fixed; change only one word. Compute all
outputs locally from disclosed fixtures. Label both as examples, never as edits
to a closed capsule.

## Composition Sequence

1. exact question;
2. direct correction;
3. five-node causal pipeline;
4. computed invented example;
5. hypothetical one-word comparison;
6. one-time selection correction;
7. time-capsule bridge;
8. participation invitation;
9. optional proof detail.
