01 · Create the numberShared method

Close one shared moment

What people leave becomes one frozen manifest. Its fingerprint joins a public Bitcoin block, and a rule places the result inside the chosen number range.

Fingerprint: a short code that changes if any record changes.

  1. Publish first. Agree on the range and mapping rule before intake closes.
  2. Freeze the whole. Public messages and saved-message commitments enter one manifest.
  3. Add a later public signal. The fingerprint joins the Bitcoin closing block.
  4. Map once. The published rule produces the starting token number.
Candidate mapping shapeminimum + (hash(fingerprint + block) mod range size)
02 · Maintain the capsuleStewardship proposal

Keep three future bundles alive

The frozen public proof can be copied widely. Saved messages need a separate protected bundle for each opening—and people responsible for keeping those bundles recoverable.

Release bundle: the saved messages and proofs due at one opening.

  1. Separate the horizons. Package 2, 5, and 10-year messages independently.
  2. Keep verified copies. A damaged or lost device must not become the end of the capsule.
  3. Split access. No single absent person should be able to lock everyone out.
  4. Practice recovery. Check copies, formats, and steward succession while there is still time to repair them.
Candidate starting point—not approvedThree independent storage locations, 3-of-5 steward access, and a yearly restore check.
03 · Open in the futureVerification path

Open only what is due

When one horizon arrives, its bundle opens. The message and its secret proof are published together, so anyone can check that the message was already inside the original capsule.

Commitment: the old fingerprint of one saved message, kept in the public manifest.

  1. Reach the date. The published opening schedule says which bundle is due.
  2. Recover access. Enough stewards reunite the access needed for that bundle.
  3. Reveal with proof. Publish the message and secret used to make its old commitment.
  4. Verify in public. Anyone can reproduce the commitment; later bundles remain closed.

What never changes

The opening adds visibility. It does not rewrite the original ritual.

  • The master manifest and fingerprint stay frozen.
  • The starting token number is produced only once.
  • Every saved reveal must match its original commitment.
  • Only the bundle whose date arrived is opened.

What still needs a decision

This is the work required before the rehearsal becomes a real capsule.

  • The final token range and exact mapping rule.
  • Encryption, archive format, and long-term file migration.
  • Who the stewards are and how they are replaced.
  • How many copies, access shares, and recovery checks are enough.

How can saved messages return without changing the fingerprint?

Each saved message gets a random secret. The message and secret produce a small digital commitment. The public capsule keeps that commitment, not the message text.

All public records and saved-message commitments become one master manifest. Its fingerprint is anchored at closing. At the 2, 5, or 10-year opening, the capsule stewards reveal the message and its secret proof. Anyone can check that they reproduce the old commitment.

The candidate shape is commitment = SHA256(capsule + opening + place + message + secret). This design does not implement or approve storage, encryption, keys, recovery, or the final ceremony rule.
