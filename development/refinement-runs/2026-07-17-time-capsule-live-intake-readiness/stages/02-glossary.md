# Local Glossary — Time Capsule Live Intake

Status: run-local explanatory vocabulary; no canonical promotion

| Term | Plain meaning | Why it matters | Misuse warning |
| --- | --- | --- | --- |
| capability state | one of static, synthetic, public-live, or sealed-live | prevents code existence from implying permission | not a maturity score or launch approval |
| envelope | private six-place contribution workspace for one credential and cohort | holds drafts and confirmed versions | not a public identity or proof of one human |
| slot | one optional word place at one horizon and position | enforces two-per-horizon quota | not token weight or contribution rank |
| confirmation version | immutable snapshot produced when a participant confirms | supports edits without rewriting prior history | not final manifest inclusion |
| disposition | accepted, rejected, quarantined, superseded, show, or withhold outcome | separates record history from display policy | quarantine is not root deletion |
| close guard | fail-closed boundary that prevents writes at/after the deadline | blocks race-based result changes | not merely a disabled button |
| public export | canonical records and receipts safe for independent verification | supplies the capsule root without private state | never includes credentials or sessions |
| sealed profile | approved construction and custody contract for future opening | defines exact security and recovery guarantees | access control alone is not sealing |
| release confirmation | current consent to display an opened word at its horizon | prevents silence from becoming consent | does not change root or supply |
| data steward | owner of storage lifecycle, access, backups, restore, and migration | makes durable data responsibility explicit | custody does not imply consent authority |
| incident owner | owner of detection, containment, notification, and recovery decisions | prevents failure handling from being improvised | not automatically the database operator |
| implementation receipt | evidence that a bounded SWU ran and was checked | allows later promotion decisions to cite work | not product, security, or launch approval |
