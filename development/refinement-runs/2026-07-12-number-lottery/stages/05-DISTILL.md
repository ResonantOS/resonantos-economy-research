# Distill Result — Number Lottery

## Run Frame

- Target context: ResonantDAO research project seeking a provenance-bearing
  community ritual and 2/5/10-year capsule without inventing token economics.
- Objective and output artifact: select the smallest coherent design unit and
  emit a decision usable by Invoke Design.
- Mode and budget: Tournament; four tracks, two evidence rounds (explorer
  synthesis, reviewer balance), one pitch-off.
- Role execution path: true registered subagents—Simon as Proposer;
  Popper/Quine as Balancers; parent reconciliation.
- Verdict: **pass for design, not validation or implementation**.

## Concept Layer Map

```text
purpose: durable community meaning and reflection
  -> value/constraint: voluntary plurality, anti-capture, no policy effect
    -> capability: ritual capsule boundary
      -> workflow: admit -> freeze -> verify -> commemorate -> revisit
        -> artifacts: manifest, cohort root, symbolic integer, event receipts
          -> operations: canonicalize, order, digest, verify, quarantine, reveal
```

The original framing jumped from purpose (“meaning”) directly to operation
(sum characters) and policy (token supply). The selected unit restores the
missing capability and workflow levels.

## Selected Smallest Coherent Unit

### Ritual Capsule Boundary Contract (RCBC)

Responsibility:

> Freeze one admitted cohort under versioned rules and emit a reproducible,
> provenance-bearing symbolic integer plus commemorative event identifiers,
> while making it impossible for ritual inputs or outputs to alter token supply,
> rewards, credit, reputation, or governance authority.

Inputs:

- frozen admission/eligibility owner reference;
- versioned public-manifest schema and canonicalization profile;
- accepted public submission artifacts;
- explicit abstention/non-participation outcomes;
- cohort close time, ordering, duplicate, moderation, and disposition rules;
- current draft-policy reference (`1_000_000_000 RES`) as an immutable external
  comparison field, not an endorsed economic value.

Outputs:

- canonical public manifest;
- ordered cohort root;
- provenance-bearing symbolic integer;
- commemorative capsule/epoch labels;
- 2/5/10-year and annual event contracts;
- inclusion, integrity, moderation-disposition, and stewardship receipts;
- explicit zero-effect receipt for supply/reward/credit/reputation/authority.

Extension boundary:

- `CommittedArtifactExtension`: may add committed-but-unopened artifacts after a
  security owner selects the protocol/threat model and the relevant fixtures
  exist.
- `EncryptedRecoveryExtension`: separately deferred; adds ciphertext recovery,
  key/nonce/custody/migration/legal obligations and is not needed for RCBC.

## Closure Tests

- Responsibility: one—freeze, derive, and commemorate without policy effect.
- Named inputs/outputs: pass.
- Abstraction level: capability/workflow boundary, not token policy or crypto
  operation.
- Hidden glue: admission, moderation, policy, security, and stewardship owners
  are explicit handles; their internals are not smuggled into RCBC.
- Future scale: cohort versioning and algorithm identifiers provide extension
  seams without implementing future mechanisms.
- Meaning loss if split further: removing the zero-effect boundary turns the
  unit into an arbitrary hash tool; removing provenance turns it into numerology.

## Recomposition Proof

RCBC recomposes upward as follows:

- purpose receives provenance, continuity, and reflection;
- anti-capture receives voluntary modes, no ranking, and zero authority effect;
- the time capsule receives frozen cohort/event contracts;
- technical verification receives canonical bytes, root, and receipts;
- token policy receives no write path—only an external draft-policy reference;
- future hidden-message work attaches through an explicit extension rather than
  contaminating the minimum public path.

Failure if wrong: the project either becomes a generic checksum with no communal
meaning or a ritual capable of silently influencing monetary/governance policy.

## Evolution Profile

- Concrete pressures: repeated cohorts, language/script variation, new stewards,
  algorithm migration, harmful-content policy, contributor absence, and later
  hidden-message demand.
- Smallest extension boundary: versioned cohort/algorithm identifiers plus
  `CommittedArtifactExtension` handle.
- Deferred mechanisms: encryption/key custody, decentralized identity,
  economic mapping, reward/credit integration, and automated publication.

## Tension Ledger

- Resolved: ritual output cannot determine supply in this run.
- Resolved: character summation is educational art, not serious mechanism.
- Resolved: public baseline is minimum; commitments/encryption are extensions.
- Stable tension: immutable cohort integrity versus later consent/moderation.
- Stable tension: founder ritual meaning versus informal status capture.
- Unresolved evidence: all nine fixtures remain unimplemented/unrun.

## Premortem

Six months later the project failed because a polished capsule ceremony was
mistaken for a safe, durable mechanism before the fixture pack existed, and the
stewardship/key/content obligations were discovered only after intake.

Guardrail: intake remains prohibited until fixture execution, source-owner
decisions, and a separate implementation task-session. This Refine plan does not
authorize a live cohort.

## Frame-Expiry Note

Rerun Distill if:

- ResonantDAO changes or reopens the planned 1B supply;
- any of the nine fixtures kills the public baseline;
- a real requirement makes hidden plaintext recovery mandatory;
- governance grants ritual participation credit/reward/authority effects;
- legal/privacy review rejects the artifact or retention model.

The first deferred complexity to revisit is the commitment construction, not
encryption or token coupling.
