## Independent Initial Position — Meadows / Precedent Gate

The synthesis is directionally disciplined, but sometimes treats a combination
of owned components as though the combined research device has an owner. The
integrated mechanism has none. `Build-from-owned` must mean that each imported
component keeps its original assumptions and bridge limits; it cannot imply an
owned effect for the assembled pipeline.

| --- | --- | --- |
| descriptive families/vector | CRediT owns multi-role description and confirmation; Holmstrom-Milgrom owns omitted-task distortion | `REVISE`; component-owned, comparison effect unowned; neither owns value, completeness, dimensions, or weights (`research.md`, Agents 2 and 3) |
| prior-record to result-link pipeline | SourceCred owns editable activity graphs; CRediT typed roles; Shapley Flow edge attribution conditional on supplied DAG | unowned integration assembled from three owned representations; none owns genuine-link adjudication or the full sequence (`research.md`, Agents 1-3) |
| investment/realization/cashing aliases | local prior dispatch owns the demotion; no enabling external owner | retain `KILL alias`; the owner is of the demotion, not a working mechanism |
| bilingual communication test | no admitted owner | locally proposed and precedent-unresolved, not `precedent-clean` |

| preventive avoided-harm claim | none admitted | `FUTURE-WORK` |
| conflict-filtered randomized review | NIH owns structural COI exclusion; Tomkins owns one masking experiment; Jecmen owns constrained random assignment | split into separate devices; neither masking nor randomization discovers hidden conflict or proves competence/fairness (`research.md`, Agents 2-3) |
| qualified lottery | HRC process owns threshold then random order; Liu owns applicant survey | split; gate validity and newcomer access remain unowned (`research.md`, Agents 3-4) |
| protected newcomer pairing | none admitted | `FUTURE-WORK` |
| shadow/consequential Goodhart protocol | diagnostic, multitask, behavioral, and audit components have distinct owners | original local protocol inspired by owned components, not an externally witnessed protocol (`research.md`, Agents 3-4) |
| person totals, history rank, failure count, fairness-by-randomness | related operators exist | split the KILL row; unsupported transformations to merit, qualification, informative value, or fairness have no owner (`research.md`, Agents 1 and 3) |

Experiments A-F need the same corrections: A is component-owned but the
superiority claim is unowned; B must separate COI, masking, and assignment; C1
gate validity/access is unowned while C2 has an operated draw precedent; D has
owned diagnostics but no owned remedy; E1 has only a visibility analogue and E2
is unowned; F has no admitted owner.

Strongest objection: the main danger is a self-reinforcing loop rather than one
bad score:

`history visibility -> review/opportunity access -> more visible results ->
reviewer/incumbent status -> power to define qualification, weights, and
replacement defenses -> more history visibility`.

Each proposed defense can become the next controlled proxy. The admitted
negative precedent is Optimism RetroPGF reviewer access: badgeholders tended to
pass badges to teammates, prompting a coworker prohibition, while prior
badgeholders and top prior recipients still participated in new badge access
(`research.md`, Agent 1). This attacks history-informed reviewer selection but
does not prove local capture or transfer an effect size.

Questions for Popper: what single preregistered outcome kills the integrated
pipeline; can every failure be blamed on a component; can C1 gate failure be
separated away to preserve C2; and what benefit-over-evidence-only comparator
and harm bound witnesses recognition utility?

Questions for Quine: distinguish source author, operating institution, local
synthesis, and local experiment as owners; define genuine link, relevant
history, qualification, informative, newcomer, and community outcome; prevent
`held` from insulating a claim; decompose the omnibus KILL row; and keep later
use observable rather than causal.

This position would strengthen only after a preregistered evidence-only versus
history-assisted fixture, a longitudinal feedback-loop witness, separate gate
and draw reports, missing prior-art cohorts, and explicit naming of every
original unwitnessed effect.

## Independent Initial Position — Quine / Definitional-Soundness Gate

The synthesis preserves the decision separation better than the earlier
scaffold, but several labels still smuggle value, fairness, or truth into
descriptive procedures. Retain the research program, kill the three economic
aliases, and require stronger definitions before any fixture is frozen.

| Candidate | Initial definitional verdict |
| --- | --- |
| descriptive families/vector | `SPLIT / REVISE`; separate activity/role description from recognized contribution; confirmation verifies declaration, not truth, importance, completeness, or causation (`research.md`, Agent 2) |
| prior-record result-link pipeline | `REVISE; definition-blocked`; define whether acceptance means admissibility, temporal relevance, necessary-condition involvement, or causal effect; agreement/DAG/visibility cannot be truth (`research.md`, Agent 3) |
| investment/realization/cashing aliases | `KILL confirmed`; stored-value and conversion implications survive negative clauses |
| bilingual communication test | `RETAIN` only as independent safety test; kill if payout, debt, balance, or entitlement beliefs increase despite clearer recall |
| informative-failure dossier | `SPLIT / REVISE; no recognition arm yet`; protocol adequacy is not high effort and downstream use is not truth (`research.md`, Agent 2) |
| preventive avoided harm | split description from recognition; without a counterfactual, kill the present recognition claim |
| randomized review | `SPLIT / REVISE`; conflict, competence, assignment, disposition, and appeal are distinct; conflict-free is not qualified (`research.md`, Agent 3) |
| qualified lottery | split and provisionally rename `filtered lottery`; eligibility satisfaction is not merit and randomness is not fairness (`research.md`, Agents 3-4) |
| newcomer pairing | `FUTURE-WORK`; newcomer, experienced, protected, duty, cost, safety, and exit are undefined |
| shadow/consequential protocol | `SPLIT / REVISE`; visible but non-consequential recognition is contradictory because visibility carries attention/status (`research.md`, Agent 4) |
| person totals, history rank, failure count, fairness-by-randomness | `KILL` and expand to role-label-as-value, agreement-as-truth, use-as-causation, and implicit ranks reconstructed from result records |

Definition-collapse tests:

1. Description to value: would a harmful, irrelevant, or incomplete act still be
   recorded? If not, description is already recognition.
2. Planning to merit: can forecast effort/complexity change recognition,
   allocation, or rank? If yes, planning has become value-by-effort.
3. Link disposition to truth: `accepted` can mean only that a bounded evidence
   claim passed declared review, not causal truth.
4. History to qualification: past records may trigger a current check, not
   satisfy it.
5. Randomness to fairness: if the eligible pool can be captured, the draw
   cannot launder the gate.
6. Recognition to reward/authority: records must not be summed, ranked, used for
   payout, future access, or governance attention.
7. Use to informativeness: later citation or mention cannot prove causation or
   value.

The six decisions are: describe an act; forecast a plan; adjudicate one bounded
result-link claim; allocate one future opportunity under separate rules; make no
reward decision here; make no authority decision here.

Questions for Meadows concern de facto rank from repeated records and
incumbent-maintained qualification gates. Questions for Popper concern a link
target independent of reviewer agreement/DAG choice, a knowledge-change witness
independent of citation, and a falsifier for bilingual safety despite improved
comprehension.

This position would relax only after explicit schemas for all six decisions,
aggregation prohibitions at storage/query/UI/use boundaries, non-circular
link-adjudication targets, and prospective thresholds for entitlement,
history-based exclusion, gate capture, implicit rank, and visible-status effects.

## Group Convergence And Remaining Tension

The three positions converge on these operative changes:

1. component ownership never supplies an integrated-effect witness;
2. A, B, C, D, and E must be decomposed before registration;
3. the current informative-failure definition is killed as circular while a
   prospective E1/E2 rebuild remains researchable;
4. `investment`, `realization`, and `cashing` remain killed as mechanism aliases
   but may be tested as independent communication stimuli;
5. PRZ-04, PRZ-05, and PRZ-06 remain separate open bets;
6. the pipeline remains a locally proposed candidate but is definition-blocked
   before a real fixture;
7. all invalid transformations must have separate KILL rows; and
8. repeated records and defense proxies need technical and longitudinal attacks,
   not only a prose no-score rule.

The unresolved group tension is the missing non-circular live-setting witness
for a genuine result link. A seeded fixture can test recovery of seeded labels,
but cannot by itself establish valid community contribution recognition.
