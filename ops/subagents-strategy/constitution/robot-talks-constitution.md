# Robot-Talks Constitution

Status: active local companion constitution
Owner: DomainSpec subagent strategy route
Authority level: repository-local constitution

## Scope

This constitution governs a dispatch group that sets `robot_talks: true`.
It is a companion to
[subagents-strategy-constitution-proposal.md](subagents-strategy-constitution-proposal.md).

## Rule RT-C1. Independence Before Discussion

Agents in a robot-talks group must first form independent positions from their
assigned angle before seeing the other agents' returns.

## Rule RT-C2. Tension Over Summary

The discussion must identify tensions, contradictions, missing evidence, and
collapse risks. A merged summary is not enough.

## Rule RT-C3. Position Preservation

Each agent's initial position and final position must be preserved in the
working folder when robot-talks output feeds a downstream synthesizer.

## Rule RT-C4. Evidence Discipline

Every shared claim must cite the source, return, or artifact it rests on. Claims
without evidence are labeled as conjecture or open residue.

## Rule RT-C5. No Extra Human Gate

Robot-talks does not create an additional human gate. The parent dispatch
strategy already owns the confirmation gate and final approval route.

## Rule RT-C6. Inventory Hook

Durable robot-talks outputs must be linked from `.arcanum/inventory/log.md`.
If they become reusable evidence, add or update an entry under
`.arcanum/inventory/entries/`.

## Rule RT-C7. No Authority By Consensus

Agreement among agents is not authority. It can support a finding only when the
source evidence, owner route, and validation boundary remain explicit.
