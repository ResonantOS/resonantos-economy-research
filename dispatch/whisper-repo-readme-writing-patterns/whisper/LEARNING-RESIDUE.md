# Learning Residue

Status: observed local lessons; not canonical author voice or general truth

## What The Run Supports

- An ordinary situation, people, action, comparison, and consequence can carry
  the same experiment meaning as protocol-first copy while giving unfamiliar
  readers a concrete entry.
- Reader task, project kind, depth strategy, operation, and policy overlay are
  different axes. Mixing them creates ambiguous preset routing.
- A polished README can still be a dangerous success when it hides a required
  boundary.
- Deterministic checks are useful for paths, placeholders, fences, markers,
  selector abstention, and lifecycle refusal. They cannot decide whether the
  reader understood, whether all limits were declared, or whether the prose is
  true and safe.
- “Update all documentation” is more honest as a coverage ledger plus bounded
  pilots than as a mass prose rewrite.
- A generated runtime can host local overlays only when ownership and
  preservation are recorded outside the generated tree.

## What This Run Does Not Support

- that story-first or action-first writing improves comprehension in general;
- that a fixed README section order is best;
- that brevity, badges, visuals, stars, conversion, or self-demo measure quality;
- that `reader_job` or `first_action_path` deserve generic schema fields;
- that the active documentation pilot is newcomer-tested; or
- that the external source's templates may be copied.
