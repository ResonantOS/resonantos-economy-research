# Whisper Composition Plan

Status: frozen bounded pilot
Source: auditor-accepted [`findings.md`](../findings.md)

## Construction Choice

The selected construction is **situation → first useful action → expected
result → limit → deeper owner route**. Formal terms follow the ordinary example
that earns them.

This is not imposed on every document. It applies to the seven active
newcomer-facing pilots named in the substrate. Authority contracts, historical
evidence, generated core files, structured state, and append-only records keep
their existing owners and receive an explicit defer or exclude disposition.

## Pilot Changes

| Surface | Reader job | Bounded change |
| --- | --- | --- |
| Root README | choose a next door | Add one copyable first question with an expected bounded answer and stop condition. |
| GUIDE | rehearse the working loop | Add a first-use path before the seven-habit reference. |
| CONTRIBUTING | make one safe contribution | Add an action/result/verification/stop path using real local checks. |
| QUERYING | ask one useful question now | Add a smallest copyable prompt before the detailed examples. |
| Prize README | understand prize and bet without jargon | Lead with one ordinary failed-idea story before the field contracts. |
| Experiment examples | choose a story close to the reader's situation | Add a route table without changing the accepted examples. |
| Validation README | inspect one known-answer case | Add a first scaffold rehearsal and preserve the no-validation boundary. |

## Preserved Boundaries

- The inventory remains the sole canonical Start Here path.
- No active pilot becomes validated newcomer pedagogy.
- A structural pass cannot become `research-pass` automatically.
- No historical, authority, generated-core, or append-only prose is rewritten.
- The new preset paraphrases external precedent and carries unresolved-license
  lineage.

## Validation Plan

1. Run preset positive, negative, selector, and lifecycle fixtures.
2. Snapshot, export, restore, and verify local Whisper overlays.
3. Check every selected pilot's local Markdown targets.
4. Generate a path-level coverage ledger with zero unclassified Markdown files.
5. Run Whisper's existing draft validator on the active prose pilots as an
   editorial signal only.
6. Run repository inventory, term, whitespace, and hook checks.
7. Record reader effect as `no-conclusion` until a preregistered held-out trial.
