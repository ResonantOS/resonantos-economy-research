## Why a sibling, not an edit

The `veritasium` arc — surface the wrong model, break it, hand over an instrument, walk the parts,
state the evidence boundary, zoom out — is what makes the piece *teach* instead of *assert*. That
pedagogy is carried entirely by the `trajectory_core` and the technique set, and both are preserved
here unchanged. What does not fit a technical governance surface is the parent's **conspiratorial,
second-person register** ("let me show you something weird", "here's the strange part", "poke at
it"). `veritasium-formal` keeps the teaching movement and swaps the voice for an authoritative,
restrained one. Use the parent for blog/Substack; use this for docs, specs, and reference surfaces.

## SCU core selection

`trajectory_core` and the **technique set** are inherited from [`veritasium`](veritasium.md)

Only `resonance_core` is overridden:

### `resonance_core` — the felt meaning (overridden)

| Field | Value |
| --- | --- |
| tone | authoritative, precise, restrained — confident without theatrics |
| voice | declarative and third-person-leaning; address the reader as "the reader"/implied, not "you"; the author states what is so, then shows it |
| style register | technical-but-readable; terms are defined on first use and then used consistently; short declaratives, no rhetorical winking |
| emotional residue | the quiet authority of an explanation that earns each claim; the reader trusts the surface because it is exact |
| value signal | rigor and traceability — every assertion either quotes a source or is marked as interpretation |
| forbidden feels | chummy, conspiratorial, hype, "something weird/strange", exclamation-driven excitement, second-person hard-sell |

## Register translation (parent → formal)

These substitutions define the variant operationally. A `veritasium-formal` draft must read like the
right column.

| Beat | `veritasium` (parent) | `veritasium-formal` |
| --- | --- | --- |
| Opening | "You think a skill is a list of instructions." | "A skill file is not a list of instructions. It is a layered control structure." |
| Misconception | "Here's the model almost everyone carries…" | "A common reading treats a skill as a set of instructions." |
| The turn | "Here's the strange part…" | "The correction." / "Stated precisely:" |
| Instrument handoff | "Don't take my word for it — poke at it." | "Inspect the artifact directly." / "The interface below is interactive." |
| Boundary | "the whole discipline" / "confident-sounding nonsense" | "the governing discipline" / "unsupported assertion" |
| Section labels | The obvious model · The turn · Don't take my word for it | Common reading · The correction · Inspect the artifact · Layer reference · Evidence boundary · In the system |

## Validation checklist (extends the parent's)

Inherits all seven of the parent's checks, with two register gates added:

8. **No second-person hard-sell and no conspiratorial framing** — no "you think", "something
   weird/strange", "poke at it", "let me show you". Open questions and direct instruction are allowed;
   winking is not.
9. **Every domain term is defined on first use, then used consistently** — no term is assumed and none
   drifts.

> Note: the misconception-first gate (parent check 1) still holds. Formal register does **not** mean
> dropping the named-wrong-model; it means stating it as "a common reading" rather than as a personal
> accusation. A formal draft that opens cold on a definition with no prior model named still fails.
